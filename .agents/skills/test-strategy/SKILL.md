# Test Strategy

Use this skill to produce focused and executable test plans.

## Procedure

1. Read the spec, implementation plan, and coverage scenarios.
2. Classify test needs using the testing matrix.
3. Prefer business behavior assertions over implementation-detail assertions.
4. Identify existing tests to extend before creating new files.
5. Define the narrowest validation commands.
6. Capture residual risk if a scenario cannot be automated in the current demo.

Use [the testing matrix](./references/testing-matrix.md).
