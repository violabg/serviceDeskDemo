# Artifact Workflow

Use this skill to keep the demo workflow traceable.

## Standard Artifacts

- `session-brief.md`
- `requirements-analysis.md`
- `clarification-questions.md`
- `spec.md`
- `task-breakdown.md`
- `implementation-plan.md`
- `test-plan.md`
- `review-report.md`
- `changed-files.md`

## Rules

- Every artifact must have a clear title and date.
- Record source inputs and assumptions.
- Record decisions separately from open questions.
- Plans require explicit approval before implementation.
- Handoffs must name the next agent and the artifacts it must read.
- Do not overwrite an approved artifact without recording what changed and why.
