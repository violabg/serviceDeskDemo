# 05 - Role management with permission matrix

**What to build:** An authorized administrator can create and edit roles and grant structured section/operation permissions through a manageable permission matrix.

**Blocked by:** 03 - Authorized dashboard entry and service-desk navigation.

**Status:** ready-for-agent

- [ ] Role management is available to users with role-management permission.
- [ ] Administrator can create roles with clear names and descriptions.
- [ ] Administrator can edit existing role names and descriptions.
- [ ] Administrator can view permissions as service-desk sections and operations.
- [ ] Administrator can grant and remove permissions on a role.
- [ ] Users without role-management permission cannot create or edit roles.
- [ ] Automated tests cover role creation, role editing, permission matrix changes, and unauthorized denial.
